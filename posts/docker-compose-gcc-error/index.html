Docker compose与gcc内存不足的错误

2019-01-05

解决docker-compose时pip intall lxml发生gcc内存不足的错误。

记忆中python的lxml库安装好像一直不是很顺利，每次环境变更都要折腾不少时间。这次又再次遇到了相似的问题。

经过本地docker-compose up测试后没有问题但是服务器pull代码运行时却出现了问题。

server OS : Docker centOS7 Application

最初只是做proxy用的所以选了尽可能小的，但是RAM如果1G以下docker都吃力所以姑且选了1G，没想到还是稍微遇到了问题

贴近问题的是这个回答

看下内存:

$ free -m
              total        used        free      shared  buff/cache   available
Mem:            991         177         574          50         238         612
Swap:             0           0           0

确实很少。。

既然物理内存不方便增加，增加swap是貌似最简单的方法了。具体方法参考digitalocean的文章

sudo fallocate -l 4G /swapfile  # 新建swap文件
sudo chmod 600 /swapfile # 限制权限
sudo mkswap /swapfile # 设置swap
sudo swapon /swapfile # 启用swap

之后

$ free -m
              total        used        free      shared  buff/cache   available
Mem:            991         182         547          50         260         607
Swap:          4095           0        4095

然后再docker-compose up就顺利跑通了。
